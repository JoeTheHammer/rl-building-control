# Experiment data

Artifacts produced by the testbed runtime are written here. The directory is
mounted into both the backend and testbed containers so experiments persist
across runs.

## Typical layout
```
data/
├── experiments/
│   ├── <suite-name>/
│   │   ├── logs/                 # Testbed stdout/stderr per PID
│   │   ├── context/              # YAML snapshots bundled for download
│   │   └── results.h5            # HDF5 file generated by ExperimentStorage
│   └── experiment_suites.db      # SQLite database with suite metadata
├── tensorboard/                  # TensorBoard log directories per suite
└── checkpoints/                  # Optional RL policy/state dict dumps
```

Exact file names may vary based on experiment configuration. The backend's
analytics module (`backend/src/services/analytics.py`) loads HDF5 data from this
folder to build charts for the UI.

Back up or prune this directory regularly; EnergyPlus simulations can generate
large datasets.
