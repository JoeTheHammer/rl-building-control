# Frontend (React + Vite)

This package implements the user interface for the RL building-control testbed.
It is responsible for configuring experiments, monitoring suite execution, and
surfacing analytics generated by the backend.

## Features
- **Dashboard & Archive** – list active experiment suites, view status, and
  browse archived runs (`src/components/experiments`).
- **Configuration editors** – YAML-backed forms for environments, controllers,
  and experiment suites (`src/components/configurator`).
- **Analytics** – visualization widgets that consume the backend analytics API
  to compare metrics across runs (`src/components/analytics`).
- **Shared services** – typed REST clients in `src/services/` encapsulate the
  backend API surface.

## Development workflow
```bash
cd frontend
npm install
npm run dev
```
The Vite dev server listens on port `5173` and proxies REST requests to the
backend URL defined by `VITE_BACKEND_URL`.

### Environment variables
Create a `.env.local` file to override the defaults:
```
VITE_BACKEND_URL=http://localhost:8000
```
Other useful flags include:
- `VITE_LOG_LEVEL` – optional logging level passed to internal helpers.
- `VITE_DEFAULT_POLL_INTERVAL` – milliseconds between status refreshes in
  experiment dashboards.

Restart `npm run dev` whenever the environment file changes.

## Building for production
```bash
npm run build
```
This produces a static bundle in `dist/`. When the container image defined in
`Dockerfile` is used, `npm run dev -- --host 0.0.0.0 --port 5173` serves the
application in development mode; adapt the `Dockerfile` if you need a production
preview server (e.g., `npm run preview`).

## Testing & linting
```bash
npm run test       # Vitest unit tests (add suites under src/__tests__)
npm run lint       # ESLint using the repository configuration
```

## Project layout
```
src/
├── app.tsx                  # React Router root and toaster setup
├── components/              # Feature modules (dashboard, configurators, analytics)
├── lib/routes.tsx           # Route registry consumed by the navbar
├── services/                # API clients wrapping backend endpoints
├── styles/                  # Shared design tokens
└── assets/                  # Static assets (logos, icons)
```

UI state is primarily handled via React Query hooks and local component state;
no global state library is required at present.

## Interaction with other services
- All HTTP calls target the backend FastAPI service under `/api/*`.
- Experiment polling frequency, analytics downloads, and YAML save operations
  mirror the backend routers defined in `backend/src/api/`.
- WebSocket streaming is not yet implemented; long-running operations rely on
  polling helpers under `src/services/experiment-service.ts`.
